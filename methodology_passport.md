Тема: «Разработка интеллектуальной информационной системы построения индивидуальной траектории обучающегося при изучении математики»

Ключевые публикации:

1\. Evaluating Mathematical Reasoning Beyond Accuracy: https://arxiv.org/abs/2404.05692

2\. Findings of MEGA: Maths Explanation with LLMs using the Socratic Method for Active Learning: https://arxiv.org/abs/2507.12079

3\. AdaptMI: Adaptive Skill-based In-context Math Instruction for Small Language Models: https://arxiv.org/abs/2505.00147

4\. Step Guided Reasoning: Improving Mathematical Reasoning using Guidance Generation and Step Reasoning: https://arxiv.org/abs/2410.19817

5\. SPL: A Socratic Playground for Learning Powered by Large Language Model: https://arxiv.org/abs/2406.13919

Современные интеллектуальные обучающие системы (например, «Яндекс Репетитор AI») преимущественно ограничиваются проверкой финального ответа, не анализируя корректность промежуточных шагов решения. Это приводит к тому, что ученик не получает обратную связь о месте и характере ошибки, что снижает эффективность обучения. Кроме того, существующие системы не учитывают историю предыдущих ошибок ученика при формировании объяснений, что делает обратную связь неадаптированной к индивидуальным затруднениям.

Практическая задача: разработать прототип интеллектуального репетитора по математике 7–9 классов, который:

1.  Проверяет промежуточные шаги решения (гибрид: символьная проверка + LLM);
2.  Учитывает историю ошибок ученика при формировании объяснений;
3.  Строит адаптивные объяснения на основе принципов когнитивной архитектуры ACT-R.

«Интеллектуальная обучающая система по математике, реализующая пошаговую верификацию решений (гибрид: символьная проверка + LLM), учет истории ошибок (последние 5 ошибок с типизацией) и адаптивные объяснения на основе принципов ACT-R (отслеживание активации продукций, акцент на ошибочных правилах), обеспечит статистически значимое повышение итоговой успеваемости (прирост тестового балла ≥10% при α=0.05) по сравнению с контрольной системой (проверка только финального ответа, аналог Яндекс Репетитор AI) при условиях 4-недельного регулярного курса (3 занятия в неделю по 30 минут) по геометрии 8 класса (на примере темы «Признаки подобия треугольников») на выборке из 30–50 учеников одной школы с сопоставимым исходным уровнем знаний (входной тест с разницей ≤5%) ».

**ОПЕРАЦИОНАЛИЗАЦИЯ ПЕРЕМЕННЫХ:**

Независимая переменная:

Система A (экспериментальная): пошаговая верификация (SymPy + LLM), учет истории ошибок, адаптивные объяснения на основе ACT-R.

Система B (контрольная): проверка только финального ответа (аналог Яндекс Репетитор AI).

Зависимая переменная:

Основная: прирост балла в итоговом тесте (процент верно решенных задач) по сравнению с входным тестом.

Вторичные:

1) количество попыток до верного решения;

2) среднее время решения задачи;

3) частота повторных ошибок одного типа.

Контрольные переменные:

1) Исходный уровень знаний (пре-тест);

2) Время работы с системой (фиксированное);

3) Технические условия.

**ПРОЦЕДУРА ФАЛЬСИФИКАЦИИ**

Гипотеза будет **опровергнута**, если выполняется хотя бы одно из условий:

1.  **Отсутствие значимого эффекта:** прирост балла при работе с системой A превышает прирост при работе с системой B **менее чем на 5%** (ниже порога практической значимости) при **p-value ≥ 0.05** (t-критерий для зависимых выборок).
2.  **Негативный эффект:** при работе с системой A наблюдается **статистически значимое ухудшение** (p < 0.05) по вторичным метрикам (увеличение времени решения, рост количества попыток, повышение фрустрации по опроснику) по сравнению с системой B.
3.  **Эффект порядка:** статистически значимое взаимодействие между порядком предъявления систем и результатами (p < 0.05), что указывает на влияние последовательности, а не самой системы.
